what about control flow and loops in JavaScript?

JavaScript does this funky thing called a control flow, which is basically how something works from top to bottom, or the first line to the last line, unless there are some bumps in the flow (these bumps can look like loops). if i were to describe what a control flow might look like in day to day life, it's like baking a cake. there's an order and process to follow, but sometimes there's some bumps, like we ran out of baking paper or eggs, so we need to use something that is "same same but different" in order to go back to the original flow. a loop is a a code that is repeated until told to stop. an example of a loop in day to day life is like a kid playing that "but why?" game. they keep repeating the same question to you until you beg for them to stop because it's (kinda) killing your soul answering it over and over.


but wait - what's the DOM exactly?

the DOM on JavaScript stands for the Document Object Model and it visually looks like a tree! the dom shows the branches that come off of the main trunk (document), which represents each part of the webpage in your browser. for example, the document leads to the HTML which is linked to the head, the body etc. basically, the DOM reminds me of the skeleton dance song.




so we've spoken visually about what the DOM looks like, but what about how it interacts? the dom of a webpage shows all the details that went into building the webpage, and when this is shown you're able to manipulate the code that makes up the webpage directly from the DOM.


lets talk arrays and objects - and how they're different regarding data.

arrays and objects both share the same purpose of storing a collection of data. however, the difference between them is that an object represents something with properties, like a car. the object would contain data regarding it's colour, size, weight etc., and it would be contained inside "{}". however, an array stores a list of values, such as colours, and is contained inside "[]".


when trying to retrieve a data from an {object} we use an "dot notation" i.e in order to get the value from our object, we would write "person.name", for the name to be returned from said object. however, when trying to retrieve data from an [array] we use an index, which always starts from 0, and not 1. so for example, if we were trying to find out the third name from an array, we would write "name[2]".


last but not least - functions and why they're so necessary.

a function is a code that has the purpose of executing specific tasks and feeding your computer an instruction, which can be used multiple times. it allows you to then break the main task down into smaller tasks, so you can focus on each area individually.
